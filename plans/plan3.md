Plan 3 — 프런트엔드 피드 & 필터 구현
====================================

## 작업 목표
- 다중 계정 피드 탐색 UX를 완성하고 계정/날짜 기반 필터 및 페이지네이션을 적용한다.
- 사용자 피드백을 반영한 상태 표시(로딩/비어 있음/에러) 경험을 정비한다.
- 데이터 패칭과 캐싱 전략을 수립하여 대용량 피드에서도 빠른 응답성을 보장한다.

## 범위 정의
- **포함**: 피드 루트 화면, 계정/날짜 필터 UI, 페이지네이션/무한 스크롤, React Query 캐싱, 기본형 테스트.
- **제외**: 게시물 상세 모달(Plan 4), 썸네일 서버 연동(Plan 6), 관리자 노출 제어(Plan 5).

## 일정 및 마일스톤
- M1 (Day 1) — 데이터 계약 정리 및 API 모킹 환경 구축.
- M2 (Day 2) — 프로필 스트립, 필터 UI 컴포넌트 1차 완성.
- M3 (Day 3) — 페이지네이션 및 상태 관리 통합, 기본 QA 수행.
